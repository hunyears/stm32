按键抖动
    现象：按键内部的金属触点并非瞬间稳定接触或断开，经历一个持续数毫秒的微小、快速弹跳过程
    特点：
        1：持续时间：这个抖动过程通常持续 5ms 到 10ms，个别情况最长可达 20ms。
        2：发生时机：按键在按下时（前沿抖动）和释放时（后沿抖动）都可能发生。
        3：随机性：抖动的次数、波形和时间都是随机的，并非每次按键都会以完全相同的方式抖动。

    解决方法：
        1：硬件消抖
            在按键电路中增加元器件来滤除抖动信号，不占用CPU资源，但会增加硬件成本。

            方法	原理	特点
            RS触发器	利用RS触发器的"记忆"功能，一旦状态翻转，后续的触点抖动不会改变输出状态。	效果好，但电路稍复杂。
            RC滤波（电容）	在按键两端并联一个电容，利用电容的充放电特性吸收和滤除电压的尖峰毛刺。	电路简单、常用。典型值：电容 0.1μF，电阻 10kΩ。
        2：软件消抖
            通过编程实现，灵活、成本低，是嵌入式开发中最常用的方法。

            方法	原理	特点
            延时法	检测到按键状态变化后，等待一段超过抖动的时间（如 10-20ms），再读取一次状态进行确认。	实现最简单，但延时期间CPU被占用，可能影响实时性。
            计数/定时器法	使用定时器中断，每隔几毫秒采样一次按键状态。只有连续多次（如10次）采样结果一致时，才确认状态有效。	不阻塞CPU，实时性好，是更专业的做法。


上拉电阻与下拉电阻
    一、问题引入：按键悬空会发生什么？

        假设你用一个按键检测"按下"和"释放"：

            VCC
             │
             │
          ┌──┴──┐
          │按键 │
          │     │
          └──┬──┘
             │
             ├───→ STM32 GPIO 引脚
             │
            ???  ← 这里悬空了！

        按键按下时：GPIO 接到 VCC，读到了高电平（确定）
        按键释放时：GPIO 没接到任何地方，悬空了！（不确定）

        悬空（浮空）的引脚会怎样？
            - 它不会自动变成 0V，也不会自动变成 3.3V
            - 它像一个"天线"，会感应周围的电磁干扰
            - 你用手靠近，读数会跳变
            - 电机转动、电源波动，读数也会变
            - 结果：GPIO 读到的值是随机的，不可预测

    二、解决办法：上拉电阻

        什么是上拉电阻？
            把一个电阻接在 GPIO 引脚和 VCC 之间。

        电路图：

                VCC (3.3V)
                  │
                  │
              ┌───┴───┐
              │ 10kΩ  │  ← 上拉电阻
              │       │
              └───┬───┘
                  │
                  ├───→ STM32 GPIO 引脚
                  │
               ┌──┴──┐
               │按键 │
               │     │
               └──┬──┘
                  │
                 GND

        工作原理：

            按键释放（断开）时：
                - GPIO 通过 10kΩ 电阻接到 VCC
                - GPIO 读到高电平（3.3V）
                - 这就是"默认状态"

            按键按下（闭合）时：
                - GPIO 直接接到 GND（按键短路）
                - 电流从 VCC → 10kΩ → 按键 → GND 流过
                - GPIO 读到低电平（0V）
                - 此时电流 = 3.3V / 10kΩ = 0.33mA（安全）

        比喻：
            上拉电阻像一个"弹簧"，默认把电平"拉"到高处。
            按键就像一个"重物"，按下时把电平"压"到低处。
            释放时，弹簧又把电平"弹"回高处。

        为什么叫"上拉"？
            因为电阻把电平"拉上去"（拉到 VCC）。

    三、下拉电阻

        什么是下拉电阻？
            把一个电阻接在 GPIO 引脚和 GND 之间。

        电路图：

                VCC
                 │
                 │
              ┌──┴──┐
              │按键 │
              │     │
              └──┬──┘
                 │
                 ├───→ STM32 GPIO 引脚
                 │
             ┌───┴───┐
             │ 10kΩ  │  ← 下拉电阻
             │       │
             └───┬───┘
                 │
                GND

        工作原理：

            按键释放（断开）时：
                - GPIO 通过 10kΩ 电阻接到 GND
                - GPIO 读到低电平（0V）
                - 这就是"默认状态"

            按键按下（闭合）时：
                - GPIO 直接接到 VCC（按键短路）
                - GPIO 读到高电平（3.3V）
                - 此时电流 = 3.3V / 10kΩ = 0.33mA（安全）

        比喻：
            下拉电阻像一个"重力"，默认把电平"拉"到低处。
            按键就像一个"电梯"，按下时把电平"抬"到高处。
            释放时，重力又把电平"拉"回低处。

        为什么叫"下拉"？
            因为电阻把电平"拉下去"（拉到 GND）。

    四、上拉 vs 下拉 对比

        类型	    默认状态	    按下状态	    适用场景
        上拉电阻	高电平（1）	低电平（0）	    按键一端接 GND
        下拉电阻	低电平（0）	高电平（1）	    按键一端接 VCC

        记忆口诀：
            上拉：默认高，按下变低（按键接 GND）
            下拉：默认低，按下变高（按键接 VCC）

    五、阻值怎么选？

        太小（如 100Ω）：
            - 按下时电流太大（3.3V / 100Ω = 33mA）
            - 可能烧坏按键或芯片
            - 功耗大

        太大（如 1MΩ）：
            - 抗干扰能力弱（阻抗太高，容易感应噪声）
            - 响应速度慢

        常用值：4.7kΩ ~ 10kΩ
            - 电流适中（0.33mA ~ 0.7mA）
            - 抗干扰能力强
            - 响应速度快

    六、STM32 内部上拉/下拉

        STM32 的 GPIO 引脚内部已经集成了上拉和下拉电阻！
        你可以通过软件配置来启用它们，不需要外部焊接电阻。

        配置方式（标准库）：

            // 内部上拉输入
            GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IPU;  // Input Pull-Up

            // 内部下拉输入
            GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IPD;  // Input Pull-Down

            // 浮空输入（无上拉下拉）
            GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IN_FLOATING;

        什么时候用内部？什么时候用外部？

            情况	            推荐做法
            普通按键	            内部上拉（省钱、省事）
            I2C 总线	            外部上拉（阻值需要精确匹配）
            开漏输出（如 DS18B20）	外部上拉（需要更强的驱动能力）
            高干扰环境	            外部上拉（阻值可选更小，抗干扰更强）

    七、实际应用举例

        例1：按键检测（内部上拉）

            电路：
                按键一端接 GPIO，另一端接 GND
                软件配置 GPIO_Mode_IPU

            效果：
                默认读到高电平（1）
                按下后读到低电平（0）

        例2：DS18B20 温度传感器（外部上拉）

            为什么必须用外部？
                - DS18B20 是开漏输出，内部晶体管只能拉低，不能拉高
                - 需要外部电阻把空闲时的电平拉到高
                - 内部上拉电阻可能阻值不合适

            阻值选择：
                - 标准值：4.7kΩ
                - 总线挂载多设备时：可减小到 2.2kΩ


温度传感器（DS18B20）
    一、它是什么？
        DS18B20 是一个"会说话"的温度计。它不需要 ADC 转换，内部自带 12 位 ADC，直接通过一根数据线把温度值（数字）告诉你。
        测温范围：-55°C ~ +125°C
        精度：±0.5°C

    二、为什么只要一根线？（核心原理）
        想象你在敲 Morse 电码：短按 = 1，长按 = 0。
        DS18B20 就是用电平的"长"和"短"来传递 0 和 1 的。

        主机（STM32）发出的信号：
            写 1：拉低 2μs，然后释放（短促的"滴"）
            写 0：拉低 60μs，然后释放（长"嗒"）

        DS18B20 回复的信号：
            回 1：我什么都不做（上拉电阻保持高电平）
            回 0：我把线拉低一会儿

        时序图（以写为例）：

            写 1：
            _____        ┌───────────────────────
                  │      │
                  └──────┘
                  2μs    释放（上拉电阻拉高）

            写 0：
            _____        ┌───────────────────────
                  │      │
                  └──────┘
                  60μs   释放

    三、电路接线图

            STM32F103                    DS18B20
           ┌─────────┐                  ┌─────────┐
           │         │                  │         │
           │     VDD ├─── 3.3V ────────┤ VDD(3)  │
           │         │                  │         │
           │         │        ┌─────────┤ DQ(2)   │
           │         │        │         │         │
           │    PAx  ├────────┤         │         │
           │   (GPIO)│        │         │         │
           │         │      4.7kΩ       │         │
           │     VDD ├───┬────┘         │         │
           │         │   │              │         │
           │     GND ├───┴── GND ───────┤ GND(1)  │
           │         │                  │         │
           └─────────┘                  └─────────┘

        为什么要有 4.7kΩ 上拉电阻？
            因为 DS18B20 的 DQ 引脚是"开漏输出"结构。
            它只能"拉低"，不能"拉高"。
            所以必须靠外部电阻把空闲时的电平"拉"到高电平。
            就像一个弹簧：没人按的时候，自动弹回高处。

    四、通信流程（比喻）
        1. STM32 喊一声："有人吗？"（拉低 480μs，然后释放）
        2. DS18B20 回应："我在！"（拉低 60~240μs 作为应答）
        3. STM32 说："跳过身份验证"（发 0xCC 命令）
        4. STM32 说："开始测温度"（发 0x44 命令）
        5. 等待 750ms（DS18B20 在测量）
        6. STM32 再次喊："有人吗？"
        7. DS18B20 回应："我在！"
        8. STM32 说："跳过身份验证"
        9. STM32 说："把数据给我"（发 0xBE 命令）
        10. DS18B20 发回 2 字节温度数据

    五、引脚定义
        引脚号	名称	功能
        1	    GND	地线
        2	    DQ	数据线（必须接上拉电阻）
        3	    VDD	电源（3.0~5.5V）


光敏传感器
    一、它是什么？
        光敏电阻是一个"光线越强，电阻越小"的特殊电阻。
        它本身不输出电压，只改变自己的阻值。
        STM32 的 GPIO 只能读 0/1（高/低电平），不能读电阻值。
        所以需要一个"分压电路"，把阻值变化变成电压变化。

    二、工作原理（核心）
        材料：硫化镉（CdS）半导体

        无光照时：
            ┌─────────────────────────┐
            │   ● ● ● ● ● ● ● ● ●    │ ← 电子被"锁"住，无法移动
            │   (载流子很少)          │
            │   电阻很大 (1MΩ以上)    │
            └─────────────────────────┘
            相当于一个"断开的开关"

        有光照时：
            ┌─────────────────────────┐
            │   ⚡⚡⚡⚡⚡⚡⚡⚡⚡    │ ← 光子撞击，电子被"激活"
            │   (载流子暴增)          │
            │   电阻很小 (几kΩ)       │
            └─────────────────────────┘
            相当于一个"导通的开关"

    三、分压电路原理（为什么需要它？）

        问题：STM32 不能直接测电阻，怎么办？
        答案：串联一个固定电阻，测"分到的电压"。

        原理图：

                 VCC (3.3V)
                    │
                    │
                ┌───┴───┐
                │ 10kΩ │  ← 固定电阻（R1）
                │       │
                └───┬───┘
                    │
                    ├────────────→ V_out（接到 STM32 的 ADC 引脚）
                    │
                ┌───┴───┐
                │光敏电阻│  ← R_LDR（阻值随光照变化）
                │       │
                └───┬───┘
                    │
                   GND

        分压公式：
                        R_LDR
            V_out = ───────────── × 3.3V
                     R1 + R_LDR

        推导过程：
            串联电路电流相等：I = VCC / (R1 + R_LDR)
            光敏电阻上的电压：V_out = I × R_LDR = VCC × R_LDR / (R1 + R_LDR)

    四、光照与电压的关系

        光照强 → 光敏电阻小 → 分到的电压少 → V_out 低 → ADC 值小
        光照弱 → 光敏电阻大 → 分到的电压多 → V_out 高 → ADC 值大

        数值例子：
        假设 R1 = 10kΩ，VCC = 3.3V

        情况1：强光，R_LDR = 1kΩ
            V_out = 1/(10+1) × 3.3 = 0.3V
            ADC值 ≈ 0.3 × 4095 / 3.3 ≈ 372

        情况2：弱光，R_LDR = 100kΩ
            V_out = 100/(10+100) × 3.3 = 3.0V
            ADC值 ≈ 3.0 × 4095 / 3.3 ≈ 3723

        情况3：黑暗，R_LDR = 1MΩ
            V_out = 1000/(10+1000) × 3.3 ≈ 3.27V
            ADC值 ≈ 4060

           │         │                │           │
           │     3.3V├────────────────┤ VCC       │
           │         │                │           │

        注：模块上已经集成了固定电阻和光敏电阻，AO 直接输出分压后的模拟电压。

    六、ADC 是什么？
        ADC = 模数转换器，把电压（模拟量）变成数字。

        STM32F103C8T6 的 ADC：
        - 分辨率：12 位（0 ~ 4095）
        - 参考电压：3.3V
        - 换算公式：ADC值 = 电压 × 4095 / 3.3

        比喻：
            ADC 就像一个"电压尺子"，把 0~3.3V 分成 4096 个刻度。
            电压 0V    → 刻度 0
            电压 1.65V → 刻度 2048
            电压 3.3V  → 刻度 4095
