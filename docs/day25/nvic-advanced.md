## 一、核心知识点

### 1.1 NVIC 深入理解

```
NVIC（嵌套向量中断控制器）：
    - 管理 Cortex-M3 的所有中断
    - 支持中断优先级和嵌套
    - 低延迟中断响应

优先级分组：
    STM32 使用 4 位表示优先级（共 16 级）
    
    分组     抢占位数  响应位数  说明
    ────────────────────────────────────
    Group 0     0        4      无抢占嵌套
    Group 1     1        3      2 级抢占
    Group 2     2        2      4 级抢占 ← 常用
    Group 3     3        1      8 级抢占
    Group 4     4        0      16 级抢占
```

### 1.2 中断嵌套

```
中断嵌套规则：
    - 高抢占优先级可以打断低抢占优先级
    - 抢占优先级相同时，响应优先级高的先执行
    - 响应优先级不能嵌套

示例（Group 2）：
    
    中断A：抢占=0，响应=0  ← 最高
    中断B：抢占=0，响应=1
    中断C：抢占=1，响应=0
    中断D：抢占=2，响应=0  ← 最低
    
    A 可以打断 B、C、D
    C 可以打断 D
    B 和 C 不能互相打断
```

### 1.3 临界区保护

```c
// 禁用中断
void Enter_Critical(void)
{
    __disable_irq();
}

// 恢复中断
void Exit_Critical(void)
{
    __enable_irq();
}

// 使用示例
void Safe_Function(void)
{
    Enter_Critical();
    // 临界区代码，不会被中断打断
    Exit_Critical();
}
```

## 二、总结

### 核心要点

```
┌─────────────────────────────────────────────────────┐
│  【优先级分组】整个程序只需设置一次                │
│  【抢占优先级】可以打断低优先级中断                │
│  【响应优先级】不能嵌套，只决定排队顺序            │
│  【临界区】禁用中断保护关键代码段                  │
└─────────────────────────────────────────────────────┘
```
